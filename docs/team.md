# 团队分工文档

项目名称：涉水审批智能审核系统  
阶段范围：节点一至节点二  
建议团队人数：4 人以内

## 成员与职责建议

### 1. Java 后端负责人

- 负责 Spring Boot 主后端接口维护
- 负责申请信息、附件信息、初审结果数据落库
- 负责后续 Java 调用 Python AI 服务的 HTTP 集成

### 2. 前端负责人

- 负责 Vue 3 页面与路由交互
- 负责申请列表、新建申请、初审结果页面优化
- 负责后续知识库检索结果与真实初审结果展示

### 3. Python AI 服务负责人

- 负责 FastAPI 服务、知识库解析与向量检索
- 负责 LangChain、ChromaDB、MCP Server 能力实现
- 负责 `knowledge_search` 与 `check_completeness` 工具维护

### 4. 数据库与文档负责人

- 负责数据库脚本、字段说明和环境配置文档
- 负责课程设计 README、部署说明、实验提交材料
- 负责整理 Git 提交记录与答辩展示材料

## 节点二协作建议

1. 先统一知识库文档目录和检查清单字段，再分别开发解析、检索和页面展示。
2. Python 负责人优先完成 PDF/DOCX 解析、文本分块、Chroma 索引与 MCP 工具。
3. Java 与前端本节点只保留接口预留，不提前实现节点三的 Agent 决策流。
4. Git 提交建议按模块拆分，至少保留知识库、MCP、文档、测试验证等不同提交记录。
